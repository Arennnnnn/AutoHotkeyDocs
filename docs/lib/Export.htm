Export [v2.1-alpha.11+]

Allows other modules to import a name defined by the current module.

Export Default FunctionDefinition
Export Default ClassDefinition
Export Variables

Parameters

FunctionDefinition
A full function definition or arrow function.
ClassDefinition
A class definition.
Variables
A list of variable names and optional initializers, with the same rules as a global declaration.

Remarks

Within each module, a single function or class export can be marked with the Default keyword to make it the module's default export. The name used in the declaration is not exported unless an additional export declaration is used. If there is no explicit default export, the module itself is the default export.

The use of an expression or variable as the default export is not currently permitted, as it could imply that the module should be executed at an earlier stage in order to appropriately bind the imports before the destination module executes. By contrast, class and function definitions are available before any module executes, and classes support self-initialization independently of module execution.

Related

Modules, Import, #Module, #Include

Examples

Simple examples.

export MyFunction() {
    return 42
}

export MyVar, MyOtherVar := 1

export default class MyModule {
    ;...
}
